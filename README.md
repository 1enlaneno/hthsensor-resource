# hthsensor-resource

## 项目简介

hthsensor-resource 是一个用于归档和发布多个独立 HTML 页面的静态资源仓库。这些页面通常对应不同的功能模块或信息展示场景，彼此之间没有强依赖关系，可以独立访问或嵌入到其他环境中使用。

本仓库不针对任何特定域名或网站，也不提供动态服务，仅作为 HTML 文件的存储与分发点，方便维护和版本管理。

## 目录说明

```
hthsensor-resource/
├── pages/           # 存放各个独立 HTML 页面文件
├── assets/          # 公共资源文件（样式、图片、脚本等）
├── README.md        # 本文件
└── LICENSE          # 仓库许可证
```

- **pages/**：每个子文件夹或文件代表一个独立的页面，命名尽量简洁，便于识别。
- **assets/**：存放页面之间可以共享的静态资源，如图标、CSS 样式或 JavaScript 库。

## 页面归档说明

所有归档页面均为独立的 HTML 文件，可直接通过浏览器打开。页面内容可能涉及数据展示、信息提示或简单的交互功能，具体用途请参考页面内注释或文档。

归档原则：
- 保持页面自包含，尽量减少外部依赖。
- 如需引用 `assets/` 下的资源，应使用相对路径。
- 页面不包含任何与具体域名绑定的链接或配置。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录或文件，并确保其可独立运行。
- 更新页面：直接修改对应文件，建议保留版本注释。
- 删除页面：确认无其他引用后，移除对应文件或目录。
- 公共资源：如需更新公共资源，请确保不影响已有页面的使用。

欢迎提交 Issue 或 Pull Request 来改进页面内容或修复问题。如有疑问，请先查看已有文档或页面注释。
